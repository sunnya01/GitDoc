##### 类属性介绍

###### 实例属性和方法内部变量的区别

实例属性是跟随着对象创建时同步在堆空间生成的，默认会给每个实例属性给初始值

方法内部变量由于生命周期只在当前方法内，因此在声明后需要显示给予初始值否则没有对应堆空间地址值的内部变量没有意义

###### 形式参数和实际参数

形式参数指的是定义代码时方法的入参名称

实际参数指的是调用方法时填入到具体入参时的内部变量名

实际参数传值如果是基本数据类型则不传递引用地址值，而是传递值的副本，方法内部实用的参数是在堆栈中新开辟的一个拷贝变量

实际参数传值如果是引用类型则方法内部的参数本质是另一个指向同一个堆地址的变量，如果中间改变量更改了指向地址，则外部实参不会受影响

###### 重载之可变个数形参

方法中定义动态个数的入参，本质上是定义一个该类型的数组，一个方法只能有一个这样的入参且要写在最后一个参数位置

如果已经有数组形式的入参则不能再定义动态个数的入参

定义语法 method(classtype(String) ...parameterName) or method( class type parameterName[] )

###### 非公开类与内部类与公开类的关系

公开类： 一个类文件下只允许有一个跟文件名相同的public类，这个就是公开类

非公开类指的是同一个类文件下非public的类，且该类不在公开类内部，此类只能通过公开类中的工厂方法去实例化提供给外部

内部类也叫成员类，指的是在类的内部再编写的类，此类可以访问外部类的所有东西，

其他包下面的类如果需要实例内部类需要通过外部类的实例对象

OutClass outclass = new OutClass();

outclass.InterClass interclass = outclass.new InterClass();
